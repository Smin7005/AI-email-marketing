From the previous git commit (2e5ad404ef305632493e89a0bc7ceed049562d7e), we have implemented a task list of onboarding user guide, but now it disappered after I click on the 'restart user guide' button. Also, when I click on that button, only the sidebar tour will refresh, but the user guide on other tabs didn't work.

## Plan:

**Bug 1 — Checklist never appeared**

`OnboardingChecklist` was defined but never placed in the component tree. Added it to `layout.tsx` inside `<NextStepWrapper>` so it renders on every dashboard page.

**Bug 2 — Checklist disappeared on restart / didn't update when tours completed**

The original `useEffect` in `OnboardingChecklist.tsx` only re-ran when `user` changed. Two things were broken: clicking restart cleared localStorage but the checklist's React state stayed stale, and completing a tour on another page set a localStorage key that the checklist had no way to detect.

The fix moves the evaluation logic into an inner `evaluate` function that runs on mount AND listens to two custom window events:

- `onboarding-reset` — dispatched from `Sidebar.tsx` when restart is clicked, causing the checklist to reappear with all steps unchecked.
- `tour-done` — dispatched from `nextstep-provider.tsx` when any tour completes or is skipped, causing the checklist to mark that step complete in real time.

**Files Changed**

- `src/app/(dashboard)/layout.tsx` — import and render `<OnboardingChecklist />` inside `<NextStepWrapper>`
- `src/components/onboarding/OnboardingChecklist.tsx` — refactor `useEffect` to register `onboarding-reset` and `tour-done` event listeners so the checklist re-evaluates without needing a page reload
- `src/components/providers/nextstep-provider.tsx` — dispatch `tour-done` custom event after `localStorage.setItem` in both `handleTourComplete` and `handleTourSkip`
- `src/components/dashboard/Sidebar.tsx` — dispatch `onboarding-reset` event after clearing localStorage in `handleRestartTour`

## Follow-up 2:

All 6 files updated. Here's the root cause and what changed:

Root cause: Every tour/checklist check had an isNewUser time-gate (7–14 days). Clicking restart only called startNextStep('sidebar-tour') directly — which bypasses the gate — but the checklist and all other page tours still checked isNewUser and silently skipped for accounts older than those windows.

Fix — onboarding-force-restart localStorage flag:

  - Sidebar.tsx — sets onboarding-force-restart = 'true' alongside clearing the other keys on restart
  - OnboardingChecklist.tsx — handleReset (fired on onboarding-reset event) now directly sets isVisible = true and clears completedSteps, bypassing the time check entirely. The regular evaluate also checks forceRestart for subsequent re-evaluations.
  - TourAutoStart.tsx — condition changed from isNewUser && !hasSeenTour to (isNewUser || forceRestart) && !hasSeenTour
  - senders/page.tsx, campaigns/create/page.tsx, campaigns/[id]/page.tsx — same condition fix in their inline useEffects

The forceRestart flag persists in localStorage until the next restart. Once a tour on a page is seen (tour-*-done set), that page won't re-trigger it — hasSeenTour guards against that.

# 2. On the Leads tab, that 'Browse Results' onboarding component should move up, also the display effect of 'Getting Started' task list is weried.

## Plan:

**Issue 1 — Browse Results tour card overlaps pagination at the bottom**

Root cause: The tour step targets `#leads-grid` (the full ag-Grid element with `domLayout="autoHeight"` spanning most of the viewport). nextstepjs with `side: 'top'` cannot find space above the tall element and falls back to bottom positioning, colliding with the pagination bar.

Fix: Add `id="leads-results-summary"` to the results summary div in `leads/page.tsx` (just above the table). Update the tour step to target `#leads-results-summary` with `side: 'bottom'`, so the tooltip appears just below the "Showing X of Y results" line — directly above the table and visually centred in the viewport.

**Issue 2 — Getting Started checklist overflows the viewport**

Root cause: The checklist card is `fixed bottom-6 right-6` with no max-height. With all 4 steps incomplete (each showing title + description), the card is tall enough to clip outside the viewport.

Fix: Add `max-h-[calc(100vh-6rem)] overflow-y-auto` to the outer div of `OnboardingChecklist.tsx` so the card scrolls internally rather than overflowing.

**Files Changed**

- `src/app/(dashboard)/leads/page.tsx` — add `id="leads-results-summary"` to the results summary div
- `src/lib/onboarding/tours.ts` — leads-tour last step: selector → `#leads-results-summary`, side → `'bottom'`
- `src/components/onboarding/OnboardingChecklist.tsx` — outer div: add `max-h-[calc(100vh-6rem)] overflow-y-auto`
