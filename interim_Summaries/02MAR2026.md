# 1. Now we need to implement a new Story in the 'onboarding flow' Epic, when user at the dashboard, first our website should introduce each tab on the side bar.

## Plan: Sidebar Navigation Tour (Onboarding Story)

### Context
The onboarding flow Epic already has page-specific tours (leads, collections, campaigns, etc.). This new story adds a `sidebar-tour` that runs first — on the `/overview` (dashboard) page — introducing each sidebar tab to new users before they dive into the individual page tours.

### Files to Modify

| File | Change |
|------|--------|
| `src/components/dashboard/Sidebar.tsx` | Add `id` attributes to each nav link so NextStep can target them |
| `src/lib/onboarding/tours.ts` | Add the new `sidebar-tour` definition |
| `src/app/(dashboard)/overview/page.tsx` | Mount `<TourAutoStart tourName="sidebar-tour" />` |

### Behaviour
- Tour fires once for new users (within 7 days of account creation) on first visit to `/overview`
- Completion/skip is persisted to `localStorage` as `tour-sidebar-tour-done = 'true'`
- Tour highlights each sidebar item sequentially with a tooltip on the right
- Uses the same `NextStepWrapper` already mounted in `src/app/(dashboard)/layout.tsx`
- No new components or providers needed — reuses existing infrastructure

### Verification
1. Clear `localStorage` in DevTools (remove all `tour-*` keys)
2. Navigate to `/overview` while logged in as a user created within 7 days
3. Confirm the tour auto-starts after ~800ms and highlights Leads → Collections → Campaigns → Senders → Analytics → Settings in order
4. Confirm skip/complete persists the `tour-sidebar-tour-done` key and the tour does not re-trigger on page reload
