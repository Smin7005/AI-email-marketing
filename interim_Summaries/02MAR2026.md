## Plan: Sidebar Navigation Tour (Onboarding Story)

### Context
The onboarding flow Epic already has page-specific tours (leads, collections, campaigns, etc.). This new story adds a `sidebar-tour` that runs first — on the `/overview` (dashboard) page — introducing each sidebar tab to new users before they dive into the individual page tours.

### Files to Modify

| File | Change |
|------|--------|
| `src/components/dashboard/Sidebar.tsx` | Add `id` attributes to each nav link so NextStep can target them |
| `src/lib/onboarding/tours.ts` | Add the new `sidebar-tour` definition |
| `src/app/(dashboard)/overview/page.tsx` | Mount `<TourAutoStart tourName="sidebar-tour" />` |

### Behaviour
- Tour fires once for new users (within 7 days of account creation) on first visit to `/overview`
- Completion/skip is persisted to `localStorage` as `tour-sidebar-tour-done = 'true'`
- Tour highlights each sidebar item sequentially with a tooltip on the right
- Uses the same `NextStepWrapper` already mounted in `src/app/(dashboard)/layout.tsx`
- No new components or providers needed — reuses existing infrastructure

### Verification

3. Confirm the tour auto-starts after ~800ms and highlights Leads → Collections → Campaigns → Senders → Analytics → Settings in order
4. Confirm skip/complete persists the `tour-sidebar-tour-done` key and the tour does not re-trigger on page reload

## Plan: Guide Book Button in Sidebar Header

### Context
A `BookOpen` icon button is added to the right of the "B2B Email Platform" header text. Clicking it resets all `tour-*-done` localStorage keys and immediately restarts the `sidebar-tour` from any page (the sidebar is always visible, so no navigation needed). The linter previously reverted the `id` attributes on nav links and the `TourAutoStart` in overview/page.tsx, so those are also re-applied here.

### Files to Modify

| File | Change |
|------|--------|
| `src/components/dashboard/Sidebar.tsx` | Re-add nav `id` attrs; add `BookOpen` button to header; wire reset + `startNextStep` |
| `src/app/(dashboard)/overview/page.tsx` | Re-add `<TourAutoStart tourName="sidebar-tour" />` |

### Behaviour
- `BookOpen` button sits to the right of the brand logo text in the sidebar header
- On click: ALL `tour-*-done` keys are removed from localStorage — this resets every onboarding story in the epic (sidebar-tour, leads-tour, collections-tour, senders-tour, create-campaign-tour, campaign-detail-tour), then `startNextStep('sidebar-tour')` is called immediately — no page navigation required
- Future-proof: any new tour added to tours.ts is automatically included in the reset without updating the handler
- `TourAutoStart` in overview/page.tsx handles the first-time auto-start for new users
