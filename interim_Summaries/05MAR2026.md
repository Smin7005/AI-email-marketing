## Plan: Restart Onboarding Guide Confirmation Dialog

### Context

The "Restart Onboarding Guide" button (BookOpen icon) in the Sidebar header currently calls `handleRestartTour()` immediately on click — clearing all tour localStorage keys and launching the sidebar tour with no warning. The goal is to show a confirmation dialog first, alerting the user that their onboarding progress will be reset, and only proceeding if they click "Yes".

### Approach

Use the existing `AlertDialog` component (`src/components/ui/alert-dialog.tsx`, built on Radix UI) to wrap the button. Radix's `AlertDialogTrigger` manages open/close state internally — no extra `useState` needed. On "Yes" confirmation, `handleRestartTour()` runs and also clears the `onboarding-checklist-dismissed` key so the checklist reappears.

## Plan: Analytics Bar Chart with Period Toggle

### Context

Below the 4 summary cards, add a bar chart showing the time-series trend of Total Campaigns, Total Emails, Sent Emails, and Open Rate. Users toggle between 1D, 7D, and 30D views. No charting library currently exists — will install recharts.

### Chart Data Design

Per time bucket (hour for 1D, day for 7D/30D): Total Campaigns = campaigns created in bucket; Total Emails = SUM(campaigns.total_recipients) for campaigns created in bucket; Sent Emails = COUNT(email_events where event_type='sent') in bucket; Open Rate = COUNT(opened)/COUNT(sent)*100 in bucket.

Chart type: recharts ComposedChart — 3 Bar components (Total Campaigns, Total Emails, Sent Emails) on left Y-axis + 1 Line component (Open Rate %) on right Y-axis.

### Files to Create/Modify

1. Install recharts via npm.

Insert a new Card block between the 4 summary cards and the 2-column section: period toggle buttons (1D | 7D | 30D) + ResponsiveContainer wrapping ComposedChart with XAxis, two YAxes, Tooltip, Legend, 3 Bars (blue/slate/green), and 1 Line (orange for Open Rate).
