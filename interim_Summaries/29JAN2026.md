# 1. Add vercel environment variable on Vercel:

● The Vercel CLI command to add an environment variable is:

  vercel env add <name>

  This will interactively prompt you for:
  
  1. The value of the variable
  2. Which environments to add it to (Production, Preview, Development)

  For non-interactive usage, you can pipe the value:

  echo "your-value" | vercel env add <name> production

  Or add for multiple environments:

  echo "your-value" | vercel env add <name> production preview development

  Other useful env commands:
  - vercel env ls - List all environment variables
  - vercel env rm <name> - Remove an environment variable
  - vercel env pull - Pull env vars to a local .env file

# 2. Verify domain name on Resend:

● Here's how to verify your Cloudflare domain on Resend:

  Step 1: Add Domain in Resend

  1. Go to https://resend.com/domains
  2. Click Add Domain
  3. Enter your domain name (e.g., yourdomain.com)
  4. Select your region (usually leave as default)
  5. Click Add

  Step 2: Get DNS Records from Resend

  Resend will show you DNS records to add. You'll typically see:
  ┌──────┬───────────────────┬───────────────────────────────────────────────────────────────┐
  │ Type │       Name        │                             Value                             │
  ├──────┼───────────────────┼───────────────────────────────────────────────────────────────┤
  │ TXT  │ resend._domainkey │ p=MIGfMA0GCSqGSIb3... (DKIM)                                  │
  ├──────┼───────────────────┼───────────────────────────────────────────────────────────────┤
  │ TXT  │ @ or root         │ v=spf1 include:amazonses.com ~all (SPF)                       │
  ├──────┼───────────────────┼───────────────────────────────────────────────────────────────┤
  │ MX   │ @ or root         │ feedback-smtp.us-east-1.amazonses.com (optional, for bounces) │
  └──────┴───────────────────┴───────────────────────────────────────────────────────────────┘
  Step 3: Add Records in Cloudflare

  1. Go to https://dash.cloudflare.com
  2. Select your domain
  3. Click DNS → Records
  4. For each record from Resend, click Add record:

  For DKIM (TXT record):
  - Type: TXT
  - Name: resend._domainkey (or whatever Resend shows)
  - Content: The long key string from Resend
  - TTL: Auto
  - Proxy: OFF (DNS only - gray cloud)

  For SPF (TXT record):
  - Type: TXT
  - Name: @
  - Content: v=spf1 include:amazonses.com ~all
  - TTL: Auto

  Step 4: Verify in Resend

  1. Go back to Resend domains page
  2. Click Verify next to your domain
  3. Wait a few minutes (DNS propagation can take up to 48 hours, but usually works within minutes on Cloudflare)

  Step 5: Update Your App

  Once verified, update your .env and Vercel environment variables:

  FROM_EMAIL=hello@yourdomain.com
