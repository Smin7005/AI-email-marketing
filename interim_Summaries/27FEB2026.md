# 1. User Guide — Onboarding Flow via NextStep (nextstepjs)

## Overview
Implement an onboarding flow and user guide using **NextStep** (nextstepjs) — a free, open-source product tour library built natively for Next.js App Router. Tours are defined in TypeScript code (no external admin UI), styled with the existing Tailwind/shadcn/ui setup, and controlled via the `useNextStep` hook.

**Why NextStep over Product Fruits:**
- Free and open source — no SaaS subscription
- Native Next.js 14 App Router support (built specifically for it)
- Tours defined in TypeScript — type-safe, code-reviewed, version-controlled
- Uses existing Tailwind + shadcn/ui — tours match the app's visual style
- Only 2 dependencies: `nextstepjs` + `motion` (Framer Motion)

**Trade-offs accepted:**
- No cloud admin UI — tour content changes require a code deploy
- No built-in analytics — tour completion tracked via `localStorage`
- Checklist widget built manually with shadcn/ui `Card` (small amount of work)

Requirements:
- Guided tours on: Leads, Collections, Campaigns flow, Senders pages
- Auto-start for new users (Clerk `createdAt` within 7 days + localStorage flag)
- Persistent floating onboarding checklist: Sender → Leads → Collection → Campaign

## Phase 3: Create the Provider Wrapper

New file: `src/components/providers/nextstep-provider.tsx`

This is a `'use client'` component wrapping children with `NextStepProvider` and `NextStep`. It also handles `onComplete` and `onSkip` callbacks to persist tour completion state in `localStorage`.

```typescript
'use client';

import { NextStepProvider, NextStep } from 'nextstepjs';
import { tours } from '@/lib/onboarding/tours';

export function NextStepWrapper({ children }: { children: React.ReactNode }) {
  const handleTourComplete = (tourName: string) => {
    localStorage.setItem(`tour-${tourName}-done`, 'true');
  };

  const handleTourSkip = (tourName: string) => {
    localStorage.setItem(`tour-${tourName}-done`, 'true');
  };

  return (
    <NextStepProvider>
      <NextStep
        steps={tours}
        onComplete={handleTourComplete}
        onSkip={handleTourSkip}
      >
        {children}
      </NextStep>
    </NextStepProvider>
  );
}
```

## Phase 6: Auto-Start Tours for New Users

Each relevant page auto-starts its tour when the user is "new" (Clerk `createdAt` within 7 days) and hasn't seen it yet (localStorage flag). Since senders, leads, and collections pages are already `'use client'`, add the hook directly. For server component pages, pass the logic into a client child component.

Pattern (add to each page's client component):

```typescript
import { useNextStep } from 'nextstepjs';
import { useUser } from '@clerk/nextjs';
import { useEffect } from 'react';

// Inside the component:
const { startNextStep } = useNextStep();
const { user } = useUser();

useEffect(() => {
  if (!user) return;
  const isNewUser = (Date.now() - (user.createdAt?.getTime() ?? 0)) < 7 * 24 * 60 * 60 * 1000;
  const hasSeenTour = localStorage.getItem('tour-senders-tour-done') === 'true';
  if (isNewUser && !hasSeenTour) {
    const timer = setTimeout(() => startNextStep('senders-tour'), 800);
    return () => clearTimeout(timer);
  }
}, [user, startNextStep]);
```

Apply the same pattern on each page with the appropriate `tour-name`.

### Auto-Start Logic

New users are identified by `user.createdAt` within 7 days (tours) / 14 days (checklist). Each tour fires once per user per browser, tracked by `localStorage`. The pattern used:

```
if (isNewUser && !localStorage.getItem('tour-X-done')) {
  setTimeout(() => startNextStep('X'), 800ms)
}
```

The 800ms delay gives the page time to fully render before the tour overlay appears.
